# 투고 → 구글시트 → 자동 입고 연동 가이드

투고 흐름: 웹사이트 투고 → 구글시트 "검토중" + Drive 투고함 저장 → 관장님이 시트에서 상태를 "진행"으로 변경 → n8n이 5분 내 감지 → 입고 폴더 투입 → 파이프라인 자동 실행.

## 1. 구글시트 만들기
1. [sheets.new](https://sheets.new) 로 새 시트 생성. 이름은 "위하다 투고 관리" 등 자유.
2. 시트 URL에서 **시트 ID**를 복사 — `https://docs.google.com/spreadsheets/d/`**`이부분이ID`**`/edit` 부분.
   (컬럼은 첫 투고가 들어올 때 자동 생성되므로 직접 만들 필요 없음)

## 2. Drive 투고함 폴더 만들기
1. 구글 드라이브에서 새 폴더 생성 (예: "위하다_투고함").
2. 폴더 URL에서 **폴더 ID**를 복사 — `https://drive.google.com/drive/folders/`**`이부분이ID`** 부분.

## 3. Apps Script 배포
1. 1번에서 만든 시트를 열고 메뉴에서 **확장 프로그램 → Apps Script** 클릭.
2. 기본 코드를 모두 지우고 `docs/google-apps-script.gs` 파일 내용을 전부 붙여넣기.
3. 코드 상단의 설정 3줄을 채우기.
   - `SHEET_ID` — 1번에서 복사한 시트 ID
   - `FOLDER_ID` — 2번에서 복사한 폴더 ID
   - `API_TOKEN` — 아무 비밀 문자열 (예: `wehada-togo-2026-비밀번호처럼`)
4. 우측 상단 **배포 → 새 배포** 클릭.
   - 유형: **웹 앱**
   - 실행 계정: **나**
   - 액세스 권한: **모든 사용자** (토큰으로 보호되므로 안전)
5. 배포하면 나오는 **웹 앱 URL**(`https://script.google.com/macros/s/.../exec`)을 복사.
   - 처음 배포 시 권한 승인 창이 뜨면 본인 계정으로 허용.

## 4. URL 입력 (두 군데)
1. **웹사이트** — `wehada-publisher/.env.local` 파일.
   ```
   GOOGLE_SCRIPT_URL=웹앱URL
   ```
2. **n8n 폴링** — `원고자동화/스크립트/.env` 파일.
   ```
   APPS_SCRIPT_URL=웹앱URL
   APPS_SCRIPT_TOKEN=3번에서 정한 API_TOKEN과 동일하게
   ```

## 5. 운영 방법
- 투고가 들어오면 시트에 **상태=검토중** 행이 쌓임. 원고 파일은 Drive 투고함에 저장됨.
- 검토 후 출간을 결정하면 해당 행의 **상태** 셀을 `진행`으로 변경 (폰에서도 가능).
- n8n이 5분 안에 감지해서 원고를 입고 폴더로 가져가고, **파이프라인** 셀에 `입고완료`가 자동 기록됨.
- 반려할 경우 상태를 `반려`로 변경 (자동화는 아무것도 하지 않음 — 기록용).
- 주의: **상태를 "진행"으로 바꾸는 행위가 곧 제작 시작**(API 비용 발생)이므로 신중하게.

## 문제 해결
- 투고가 시트에 안 들어옴 → `.env.local`의 GOOGLE_SCRIPT_URL 확인, Apps Script 재배포 시 URL이 바뀌었는지 확인.
- "진행"으로 바꿨는데 입고가 안 됨 → 스크립트/.env의 APPS_SCRIPT_URL·TOKEN 확인. 토큰이 Apps Script의 API_TOKEN과 정확히 같아야 함.
- Apps Script 코드를 수정했다면 **배포 → 배포 관리 → 수정(연필) → 새 버전**으로 재배포해야 반영됨.
